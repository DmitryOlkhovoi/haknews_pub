There's Math.random(), and then there's Math.random() (2015)

В статье обсуждается функция Math.random() в JavaScript, которая используется для генерации случайных чисел. Основное внимание уделяется реализации этого метода в движке V8 и особенностям его работы, основанным на псевдослучайных генераторах чисел (PRNG). Объясняется, что из-за ограниченной внутренней структуры, используемой в предыдущей реализации, числа могли повторяться, что плохо сказывалось на качестве генерации. Ключевым моментом является переход на более современный и качественный алгоритм xorshift128+, который обеспечивает гораздо более длительный период генерации и высокое качество случайных значений. Однако подчеркивается, что даже с улучшенной реализацией функция не является криптографически безопасной, что ограничивает ее применение в задачах, связанных с безопасностью, таких как шифрование. Вместо этого рекомендуется использовать Web Cryptography API для следующих операций, требующих большей степени защиты.

TKey – Security for the New World

TKey представляет собой новое поколение USB-ключей безопасности, выполненных в формате открытого программного и аппаратного обеспечения. Это устройство обеспечивает высокую степень конфиденциальности пользователей благодаря уникальному подходу к созданию ключевого материала, который зависит от фактора уникальности как самого устройства, так и от конкретного приложения. Устройства TKey являются результатом стремления компании Tillitis к открытости и надежности, предоставляя пользователям прозрачность в работе с их данными.

OpenLDK: A Java JIT compiler and runtime in Common Lisp

OpenLDK — это компилятор Just-In-Time (JIT) и среда выполнения для Java, полностью реализованные на языке Common Lisp. Этот проект предлагает уникальный подход к интеграции Java и Lisp, позволяя переводить байт-код Java в Lisp-код и компилировать его в машинный код для выполнения. Благодаря этому пользователи могут беспрепятственно использовать Java-библиотеки в среде Common Lisp, что делает OpenLDK идеальным решением для разработчиков, которые хотят взаимодействовать с Java-кодом, оставаясь при этом в лексиконе Lisp. Хотя OpenLDK не предназначен для высокопроизводительного выполнения Java-программ, он предлагает полезный инструмент для использования отдельных Java-библиотек в Lisp-ориентированных проектах, минимизируя при этом необходимость в внешней Java-среде выполнения. С поддержкой интеграции классов Java в систему объектов Common Lisp (CLOS), проект предоставляет ряд возможностей для разработчиков, стремящихся к упрощению своей работы с библиотеками Java в Lisp.
