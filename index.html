Google gifts a Free AI Coding Assistant to the developer community

На днях Google представил новую бесплатную версию своего инструмента для завершения кода и помощи разработчикам — Gemini Code Assist. Этот инструмент предназначен как для индивидуального использования, так и для интеграции с GitHub, где он автоматически проверяет код на наличие ошибок и предлагает рекомендации. Gemini Code Assist для разработчиков позволяет взаимодействовать с AI-моделью Google через чат, где можно задавать вопросы и получать помощь по редактированию кода. Ключевым преимуществом нового ассистента являются его впечатляющие лимиты использования: пользователи могут получать до 180,000 завершений кода в месяц, что существенно превышает аналогичные предложения других сервисов. Инструмент интегрируется с популярными средами разработки и поддерживает множество языков программирования, что делает его весьма удобным для разработчиков, работающих с различными проектами. Google стремится привлечь молодых разработчиков к использованию своего инструмента, надеясь, что в дальнейшем они могут перейти на платные версии для бизнеса.

Увольнения затрагивают не только работающих специалистов, но и будущие поколения кадров, нужных для поддержки экосистем и противодействия стихийным бедствиям.

RoboPianist: Dexterous Piano Playing with Deep Reinforcement Learning (2023)

RoboPianist — это проект, в рамках которого разрабатываются антропоморфные роботизированные руки, обучающиеся играть на пианино с использованием методов глубокого обучения с подкреплением. Transparency in Hardware/Software Interfaces

Документ RFD 552 под названием "Прозрачность в интерфейсах аппаратного и программного обеспечения" посвящен важности открытости и доступности документации для интерфейсов, связывающих аппаратные и программные компоненты в вычислительных системах. Авторы утверждают, что для разработки качественного программного обеспечения, взаимодействующего с аппаратным обеспечением, необходимо тщательное понимание этих интерфейсов. Исходя из своей философии, компания Oxide подчеркивает, что наличие прозрачной документации приводит к улучшению разработки и интеграции, а также способствует созданию экосистемы программного обеспечения вокруг аппаратного обеспечения. Важным аспектом статьи является классификация уровней прозрачности интерфейсов, начиная от полностью открытой и документированной до частной и обремененной, а также описаны распространенные аргументы против прозрачности, такие как страх копирования конкурента или дополнительные трудности с поддержкой. Авторы утверждают, что аргументы против прозрачности, основанные на подобных опасениях, часто не обоснованы и что открытость в долгосрочной перспективе ведет к более robust системам и успешным бизнес-моделям для производителей аппаратного обеспечения.

Simulating Time in Square-Root Space

Статья представляет собой обсуждение нового исследования, посвященного симуляции времени в среде многопоточных машин Тьюринга с использованием существенно меньшего объема памяти. Согласно исследованиям, описанным в статье, для всех функций времени т(n), превышающих n, возможно выполнить симуляцию с помощью пространства, равного O(√(t log t)). Это значительно лучше предыдущих результатов, полученных более 50 лет назад. Авторы работы утверждают, что их симуляция не только упрощает работу многопоточных машин, но и способствует более глубокому пониманию сложных вычислительных проблем, таких как проблема $P$ против $PSPACE$. Они также обнаружили, что ограниченные по размеру фан-ин схемы можно оценивать с использованием пространства, пропорционального размеру схемы. Данные результаты могут иметь значительные последствия для теории вычислений и практических применений в области алгоритмов.

ForeverVM: Run AI-generated code in stateful sandboxes that run forever

ForeverVM — это API для выполнения кода, который позволяет безопасно запускать произвольный код на Python в удаленном песочнице. Уникальность этого инструмента заключается в отсутствии концепции сессии: вместо этого ForeverVM использует снимки памяти, позволяя хранить неактивные машины на длительное время без потребления вычислительных ресурсов. Это делает его эффективным и масштабируемым решением для разработки приложений, которые могут обрабатывать код без необходимости управления жизненным циклом сессий. Пользователи могут взаимодействовать с ForeverVM через интерфейс REPL, что обеспечивает удобство и простоту работы. В дополнение к этому, существует командная строка (CLI) и API, которые позволяют создавать машины и выполнять код на Python и JavaScript. ForeverVM также можно интегрировать с другими клиентами, такими как Claude Desktop. Это решение подходит как для разработчиков, так и для корпоративного использования, благодаря возможностям развертывания в облаке.

Show HN: Tach – Visualize and untangle your Python codebase

Tach — это инструмент на Python, предназначенный для визуализации и обеспечения соблюдения зависимостей и интерфейсов, написанный на Rust. Он вдохновлён концепцией модульной архитектуры, позволяя разработчикам детально контролировать и управлять зависимостями между модулями в их проектах. Tach предлагает установить строгие правила, согласно которым импорты могут происходить только из объявленных зависимостей, а вызовы между модулями должны быть ограничены публичными интерфейсами. Tach легко устанавливается через pip и не оказывает влияния на время выполнения, что позволяет интегрировать его в существующие системы с минимальными затратами. Инструмент также допускает поэтапное внедрение, что особенно удобно для проектов с большой кодовой базой. С помощью Tach пользователи могут не только контролировать зависимости, но и визуализировать их с помощью графиков, а также использовать возможности интеграции с CI/CD и различными инструментами разработки.

The FFT Strikes Back: An Efficient Alternative to Self-Attention

Статья под названием "The FFT Strikes Back: An Efficient Alternative to Self-Attention" исследует новый подход к механизмам самовнимания, которые традиционно имеют квадратичную сложность, что сдерживает их применение для длинных последовательностей. Авторы предлагают метод FFTNet, основанный на Быстром Преобразовании Фурье (FFT), который позволяет эффективно обрабатывать данные с глобальным смешиванием токенов за время $\mathcal{O}(n\log n)$. Используя преобразование входных данных в частотную область, FFTNet способен устранять ограничивающие факторы традиционного самовнимания, эффективно улавливая дальнодействующие зависимости. В статье также представлены экспериментальные результаты, показывающие превосходство FFTNet над обычными моделями самовнимания и фиксированными фурье-моделями на популярных бенчмарках, таких как Long Range Arena и ImageNet.
